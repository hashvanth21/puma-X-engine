# Plan 07-03: Feedback UI & End-to-End Integration

<objective>
Build the structured feedback collection UI component and integrate interaction tracking + feedback triggers into the match result screen. Wire up the complete data pipeline from user action through to database persistence.
</objective>

<must_haves>
- FeedbackModal component with 3 structured questions (fit, use case, style preference)
- Premium Framer Motion animations matching the existing PUMA design language
- Feedback modal triggered after recommendation view with appropriate delay
- Interaction events fired for: clicked_primary, opened_alternatives, time_spent
- Event buffer flushed on screen transition and feedback submission
- Full end-to-end flow: user sees recommendation → interactions tracked → feedback collected → all persisted to Supabase
</must_haves>

<action>
Create `puma-frontend/src/components/FeedbackModal.tsx`:

The component should:
1. Accept props: `isOpen`, `onClose`, `recommendationId`, `sessionId`, `shoeModel` (the recommended shoe name) 
2. Show 3 structured feedback questions using chip/radio selection:

**Question 1 — How did it fit?**
Options as selectable chips:
- ✅ Perfect fit
- ↔️ Slightly tight
- ❌ Too tight
- ↔️ Slightly loose
- ❌ Too loose

**Question 2 — Right for your purpose?**
Options:
- 👍 Good for my purpose
- 👎 Not ideal

**Question 3 — Style preference?**
Options:
- ❤️ Liked the style
- 💔 Didn't like the style

3. Submit button POSTs to `/api/feedback` with:
```json
{
  "recommendation_id": "...",
  "session_id": "...",
  "fit_feedback": "perfect_fit",
  "use_case_feedback": "good_for_purpose",
  "style_feedback": "liked_style"
}
```

4. On success, show a brief "Thank you" message (1.5s) then auto-close
5. On dismiss (X button), close without submitting

**Design requirements:**
- Use glassmorphism background (backdrop-blur, semi-transparent bg) consistent with existing PUMA screens
- Framer Motion `AnimatePresence` for entry/exit (slide up + fade)
- Chip buttons use the existing design token colors (puma-red accent for selected state)
- Mobile-responsive: stack questions vertically on small screens
- Non-blocking: user can dismiss at any time
</action>

<task id="07-03-02" type="code">
<title>Integrate tracking and feedback into match result screen</title>
<read_first>
- puma-frontend/src/screens/Screen4Match.tsx (or the current match result screen — read the full file to understand structure)
- puma-frontend/src/stores/index.ts (AppStore with analytics slice — trackInteraction, flushEvents, sessionId, recommendationId)
- puma-frontend/src/components/FeedbackModal.tsx (from previous task)

<action>
Modify the match result screen (Screen4Match.tsx or equivalent) to:

1. **On mount:** Track `time_spent` by starting a timer. On unmount or screen transition, fire `trackInteraction('time_spent', { duration_ms: elapsed })`.

2. **On recommendation load:** Store the `recommendationId` from the backend response into the analytics store via `setRecommendationId(response.recommendationId)`.

3. **On primary shoe click:** Fire `trackInteraction('clicked_primary', { model: shoe.model })`.

4. **On alternatives section open/expand:** Fire `trackInteraction('opened_alternatives')`.

5. **Feedback trigger:** After recommendation is displayed for 5 seconds, show a subtle inline prompt (small banner below the result) that says "How was this recommendation?" with a "Give Feedback" button. Clicking it opens the FeedbackModal.

6. **On feedback submit or screen transition:** Call `flushEvents()` to send buffered interaction events to the backend.

7. **Import and render FeedbackModal** with AnimatePresence, passing `isOpen`, `onClose`, `recommendationId`, `sessionId`.

Example integration points:
```tsx
// On mount — time tracking
useEffect(() => {
  const start = Date.now();
  return () => {
    trackInteraction('time_spent', { duration_ms: Date.now() - start });
    flushEvents();
  };
}, []);

// Delayed feedback prompt
useEffect(() => {
  const timer = setTimeout(() => setShowFeedbackPrompt(true), 5000);
  return () => clearTimeout(timer);
}, []);
```
</action>

- Primary shoe click fires trackInteraction('clicked_primary')
- Alternatives section fires trackInteraction('opened_alternatives')
- Feedback prompt appears after 5-second delay
- FeedbackModal renders with correct props (recommendationId, sessionId from store)
- flushEvents() called on screen transition (cleanup effect)
- recommendationId from backend response is stored in analytics slice
</acceptance_criteria>
</task>

```typescript
const response = await fetch(`${API_BASE}/api/recommend`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    footProfile,
    context,
    session_id: sessionId,  // <-- ADD THIS
  }),
});
const data = await response.json();

// Store recommendation AND the tracking ID
setRecommendation(data.recommendation);
setRecommendationId(data.recommendationId);  // <-- ADD THIS
```

Also add `beforeunload` listener in the app root (App.tsx or main entry) to flush remaining events:

```typescript
useEffect(() => {
  const handleUnload = () => {
    const events = useAppStore.getState().interactionBuffer;
    if (events.length > 0) {
      navigator.sendBeacon(
        `${API_BASE}/api/events`,
        JSON.stringify({ events })
      );
    }
  };
  window.addEventListener('beforeunload', handleUnload);
  return () => window.removeEventListener('beforeunload', handleUnload);
}, []);
```
</action>
