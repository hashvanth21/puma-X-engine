# Plan 04-02: Weighted Scoring Algorithm & Recommendation Engine

## Objective
Build the rule-based weighted scoring engine that takes a foot profile + user context, scores all shoes in the catalog, and produces a ranked result with primary recommendation (highest score), alternate recommendation (second highest), and a list of eliminated shoes. The engine is implemented as a pure TypeScript function on the frontend and mirrored as a backend service.

## must_haves
- Engine produces primary + alternate recommendation for any valid input combination (REC-01)
- Primary shoe has a match percentage (0-100) (REC-02)
- Alternate shoe has a match percentage (REC-03)
- Engine uses rule-based matching with weighted scoring against shoe metadata attributes (REC-06)
- Scoring is deterministic — same inputs always produce same output
- Engine works as pure function with no side effects

```typescript
  useCase: 0.25,
  activity: 0.20,
  climate: 0.10,
  priority: 0.10,
  wearDuration: 0.05,
};

export function scoreShoe(
  shoe: Shoe,
  footProfile: FootProfile,
  context: UserContext,
  weights?: ScoringWeights
): number;

export function generateRecommendation(
  footProfile: FootProfile,
  context: UserContext,
  shoes: Shoe[],
  weights?: ScoringWeights
): Recommendation;
```

**scoreShoe algorithm:**

1. **Foot Profile Score (0-10):**
   ```
   widthScore = shoe.attributes.wideFootScore (if foot is 'wide') 
                OR 10 - shoe.attributes.wideFootScore (if foot is 'narrow')
                OR 7 (if foot is 'standard')
   archMatch = shoe.attributes.archSuitability.includes(foot.arch) ? 10 : 3
   pronationMatch = shoe.attributes.pronationSupport.includes(foot.pronation) ? 10 : 2
   footProfileScore = (widthScore + archMatch + pronationMatch) / 3
   ```

2. **Use Case Score (0-10):**
   Map `context.useCase` to relevant shoe score:
   - `'daily-commute'` → shoe.attributes.dailyCommuteScore
   - `'running'` → shoe.attributes.runningScore
   - `'long-standing'` → shoe.attributes.longWearComfortScore
   - `'gym-casual'` → (shoe.attributes.dailyCommuteScore + shoe.attributes.longWearComfortScore) / 2
   - `'office-wear'` → shoe.attributes.dailyCommuteScore * 0.8 + shoe.attributes.longWearComfortScore * 0.2
   - `'rainy-conditions'` → shoe.attributes.wetGripScore

3. **Activity Score (0-10):**
   Map `context.activity`:
   - `'running'` → shoe.attributes.runningScore
   - `'walking'` → (shoe.attributes.dailyCommuteScore + shoe.attributes.longWearComfortScore) / 2
   - `'standing'` → shoe.attributes.longWearComfortScore
   - `'gym-casual'` → (shoe.attributes.runningScore * 0.3 + shoe.attributes.dailyCommuteScore * 0.7)

4. **Climate Score (0-10):**
   - If `context.climate === 'rainy'`:
     - If shoe.attributes.idealEnvironment includes 'rainy': shoe.attributes.wetGripScore
     - Else: shoe.attributes.wetGripScore * 0.6 (penalty)
   - If `context.climate === 'dry'`: 8 (neutral — most shoes work in dry)

5. **Priority Score (0-10):**
   - `priorityScore` is 0-100 from slider
   - If priorityScore < 33 (comfort): shoe.attributes.longWearComfortScore
   - If priorityScore > 66 (performance): (shoe.attributes.raceUseScore + shoe.attributes.runningScore) / 2
   - Else (balanced): (shoe.attributes.longWearComfortScore + shoe.attributes.runningScore) / 2

6. **Wear Duration Score (0-10):**
   - `hours = context.hoursPerDay`
   - If hours >= 8: shoe.attributes.longWearComfortScore * 1.0 + (shoe.attributes.weight < 300 ? 1 : -1)
   - If hours >= 5: (shoe.attributes.longWearComfortScore + 5) / 2
   - Else: 7 (most shoes fine for < 5 hours)

**Total:** `totalScore = Σ(dimensionScore × weight)` where weights sum to 1.0.
**Match percentage:** `Math.round((totalScore / 10) * 100)`, clamped to 0-100.

**generateRecommendation:**
1. Score all shoes → sort descending
2. Primary = shoes[0], alternate = shoes[1]
3. Eliminated = shoes.slice(2) (take 3-4 for display, not all)
4. Generate empty matchReasons and eliminatedReasons arrays (Plan 04-03 fills these)
5. Return `Recommendation` object with `generatedAt: new Date().toISOString()`

```typescript
import { Router, Request, Response } from 'express';
import { SHOES_CATALOG } from '../data/shoes';
import { generateRecommendation, RecommendRequest } from '../services/scoringEngine';

const router = Router();

// POST /api/recommend — accept foot profile + context, return recommendation
router.post('/', (req: Request, res: Response) => {
  const { footProfile, context } = req.body as RecommendRequest;

  if (!footProfile || !context) {
    res.status(400).json({ error: 'Missing footProfile or context in request body' });
    return;
  }

  // Validate required fields
  if (!footProfile.width || !footProfile.arch || !footProfile.pronation) {
    res.status(400).json({ error: 'footProfile must include width, arch, and pronation' });
    return;
  }

  if (!context.useCase || !context.activity || !context.climate || context.priorityScore === undefined) {
    res.status(400).json({ error: 'context must include useCase, activity, climate, and priorityScore' });
    return;
  }

  const recommendation = generateRecommendation(footProfile, context, SHOES_CATALOG);
  res.json({ recommendation });
});

export { router as recommendRouter };
```

The endpoint:
1. Validates request body has required fields
2. Calls `generateRecommendation` with the full shoe catalog
3. Returns the `Recommendation` object
