# Plan 04-03: Explanation Generator — Match Reasons & Elimination Logic

## Objective
Build the explanation generator that produces human-readable, PUMA-branded match reasons for the primary recommendation and specific elimination reasons for disqualified shoes. Integrate the generator into the recommendation engine so that `generateRecommendation` returns fully populated `primaryReasons`, `alternateReasons`, and `eliminatedShoes` arrays with structured explanation text that references specific foot profile attributes and context inputs.

## must_haves
- Explanation text is specific: references foot profile attributes and context (REC-04)
- "Why not the others" section lists at least 2 eliminated models with reasons (REC-05)
- Primary recommendation has 3+ match reasons
- Elimination reasons reference specific attribute deficiencies
- No generic text like "this shoe scored higher" — must cite biomechanical attributes

**Exports:**
```typescript
import type { Shoe, FootProfile, UserContext, MatchReason } from '@/types';

export function generateMatchReasons(
  shoe: Shoe,
  footProfile: FootProfile,
  context: UserContext
): MatchReason[];

export function generateEliminationReason(
  shoe: Shoe,
  winner: Shoe,
  footProfile: FootProfile,
  context: UserContext
): string;
```

**generateMatchReasons implementation:**

Produce 3-5 `MatchReason` objects by checking which dimensions scored highest for this shoe:

1. **Width Match** (if foot is 'wide' and shoe wideFootScore >= 7):
   ```
   { attribute: "Foot Width", explanation: "Your wider foot profile gets room to breathe in the ${shoe.model}'s spacious forefoot — no pinching during long ${context.activity} sessions", positive: true }
   ```

2. **Width Mismatch** (if foot is 'wide' and shoe wideFootScore < 4):
   ```
   { attribute: "Foot Width", explanation: "The ${shoe.model} runs narrow — tight fit risk for your wider foot", positive: false }
   ```

3. **Arch Support** (if shoe.archSuitability includes foot.arch):
   ```
   { attribute: "Arch Support", explanation: "The ${techFeature} in the ${shoe.model} matches your ${foot.arch} arch profile perfectly", positive: true }
   ```

4. **Pronation Match** (if shoe.pronationSupport includes foot.pronation):
   ```
   { attribute: "Pronation Control", explanation: "Built-in ${foot.pronation === 'overpronation' ? 'stability guidance' : 'neutral platform'} keeps your stride aligned mile after mile", positive: true }
   ```

5. **Climate Match** (if context.climate === 'rainy' and shoe.wetGripScore >= 6):
   ```
   { attribute: "Wet Weather Grip", explanation: "The ${shoe.techFeatures.includes('PUMAGRIP') ? 'PUMAGRIP outsole' : 'rubber outsole'} maintains traction on wet surfaces — essential for your rainy ${context.useCase.replace('-', ' ')} route", positive: true }
   ```

6. **Comfort for Long Wear** (if context.hoursPerDay >= 7 and shoe.longWearComfortScore >= 7):
   ```
   { attribute: "All-Day Comfort", explanation: "With ${context.hoursPerDay}+ hours on your feet, the ${shoe.techFeatures[0]} cushioning in the ${shoe.model} won't compress or bottom out", positive: true }
   ```

7. **Performance Match** (if context.priority === 'performance' and shoe.raceUseScore >= 7):
   ```
   { attribute: "Race Performance", explanation: "The ${shoe.techFeatures.find(t => t.includes('PWRPLATE')) || 'responsive midsole'} converts your energy into forward propulsion — built for speed, not just comfort", positive: true }
   ```

8. **Use Case Alignment** (always include — map useCase to relevant explanation):
   ```
   'daily-commute' → "Designed to handle the stop-start of daily commuting — from train platforms to office floors"
   'running' → "Engineered for the rhythmic impact of distance running — ${shoe.attributes.runningScore}/10 running suitability"
   'long-standing' → "Extra cushioning depth absorbs the compression of standing for ${context.hoursPerDay}+ hours daily"
   'gym-casual' → "Versatile enough for gym workouts and casual outings without swapping shoes"
   ```

Filter to only positive reasons, cap at 5, take at least 3. If fewer than 3 positive reasons exist, include the top 3 by relevance (arch, pronation, use case always apply).

**generateEliminationReason implementation:**

Determine the PRIMARY reason this shoe was eliminated by finding its weakest dimension relative to the user's needs:

1. If foot is 'wide' and shoe.wideFootScore < 4: `"${shoe.model}: Too narrow for your wider foot profile — ${shoe.attributes.wideFootScore}/10 width accommodation"`
2. If context.climate === 'rainy' and shoe.wetGripScore < 4: `"${shoe.model}: Low wet-grip rating (${shoe.attributes.wetGripScore}/10) — not safe for your rainy ${context.useCase} conditions"`
3. If shoe.category === 'lifestyle' and context.activity === 'running': `"${shoe.model}: A lifestyle sneaker — insufficient cushioning and support for ${context.hoursPerDay}+ hours of running"`
4. If foot.pronation === 'overpronation' and !shoe.pronationSupport.includes('overpronation'): `"${shoe.model}: No overpronation support — your gait pattern needs guidance that this shoe doesn't provide"`
5. If context.priority === 'performance' and shoe.raceUseScore < 3: `"${shoe.model}: Comfort-focused design lacks the responsive energy return your performance goals demand"`
6. If context.hoursPerDay >= 8 and shoe.longWearComfortScore < 5: `"${shoe.model}: Cushioning breaks down during extended ${context.hoursPerDay}-hour wear sessions"`
7. Fallback: `"${shoe.model}: The ${winner.model} outperforms on ${findWeakestDimension(shoe, winner, context)} — the deciding factor for your profile"`

The function `findWeakestDimension` compares the two shoes on the user's most relevant dimensions and returns a human-readable label (e.g., "wet-weather traction", "all-day cushioning", "arch support").

```typescript
  primary: primary.shoe,
  primaryMatchScore: primary.score,
  primaryReasons,
  alternate: alternate.shoe,
  alternateMatchScore: alternate.score,
  eliminatedShoes,
  generatedAt: new Date().toISOString(),
} satisfies Recommendation;
```
