# Plan 02-02: Deterministic Metrics Generation

## Objective
Convert the final 'complete' status of the scan state machine into physical output cards. The user must feel like complex biomechanical processing just happened. 

## Tasks

<task id="02-02-T1">
<title>Create standard Metric/Data Card component</title>
<action>
Create `puma-frontend/src/components/MetricCard.tsx` wrapping a Framer Motion container to display individual metrics (like "Width", "Arch Type").
It should render beautifully inside our GlassCard theme.
</action>
</task>

<task id="02-02-T2">
<title>Build ScanResults modal view</title>
<action>
Create `puma-frontend/src/screens/Screen2FootScan/ScanResults.tsx`.
When the scan completes, render this component which loops through our mocked metrics and surfaces them sequentially using `staggerContainer`:
- Foot Width: Standard
- Arch Type: Medium
- Pressure Distribution: Forefoot strike
- Estimated Size: 43 EU

It should dispatch to Zustand `setFootProfile` asynchronously immediately upon mount, so the rest of the application captures it.
</action>
</task>
